GstElementClass

GstElement is the abstract base class needed to construct an element that can be used in a GStreamer pipeline. Please refer to the plugin writers guide for more information on creating Gst.Element subclasses.

The name of a Gst.Element can be get with gst_element_get_name (not introspectable) and set with gst_element_set_name (not introspectable). For speed, GST_ELEMENT_NAME (not introspectable) can be used in the core when using the appropriate locking. Do not use this in plug-ins or applications in order to retain ABI compatibility.

Elements can have pads (of the type Gst.Pad). These pads link to pads on other elements. Gst.Buffer flow between these linked pads. A Gst.Element has a GLib.List of Gst.Pad structures for all their input (or sink) and output (or source) pads. Core and plug-in writers can add and remove pads with Gst.Element.prototype.add_pad and Gst.Element.prototype.remove_pad.

An existing pad of an element can be retrieved by name with Gst.Element.prototype.get_static_pad. A new dynamic pad can be created using Gst.Element.prototype.request_pad with a Gst.PadTemplate. An iterator of all pads can be retrieved with Gst.Element.prototype.iterate_pads.

Elements can be linked through their pads. If the link is straightforward, use the Gst.Element.prototype.link convenience function to link two elements, or gst_element_link_many (not introspectable) for more elements in a row. Use Gst.Element.prototype.link_filtered to link two elements constrained by a specified set of Gst.Caps. For finer control, use Gst.Element.prototype.link_pads and Gst.Element.prototype.link_pads_filtered to specify the pads to link on each element by name.

Each element has a state (see Gst.State). You can get and set the state of an element with Gst.Element.prototype.get_state and Gst.Element.prototype.set_state. Setting a state triggers a Gst.StateChange. To get a string representation of a Gst.State, use Gst.Element.prototype.state_get_name.

You can get and set a Gst.Clock on an element using Gst.Element.prototype.get_clock and Gst.Element.prototype.set_clock. Some elements can provide a clock for the pipeline if the Gst.ElementFlags.prototype.provide_clock flag is set. With the Gst.Element.prototype.provide_clock method one can retrieve the clock provided by such an element. Not all elements require a clock to operate correctly. If the Gst.ElementFlags.prototype.require_clock() flag is set, a clock should be set on the element with Gst.Element.prototype.set_clock.

Note that clock selection and distribution is normally handled by the toplevel Gst.Pipeline so the clock functions are only to be used in very specific situations.

Gst.Element.prototype.get_state 

function Gst.Element.prototype.get_state(timeout: int ): {
    // javascript wrapper for 'gst_element_get_state'
}

Gets the state of the element.

For elements that performed an ASYNC state change, as reported by Gst.Element.prototype.set_state, this function will block up to the specified timeout value for the state change to complete. If the element completes the state change or goes into an error, this function returns immediately with a return value of Gst.StateChangeReturn.prototype.success or Gst.StateChangeReturn.prototype.failure respectively.

For elements that did not return Gst.StateChangeReturn.prototype.async, this function returns the current and pending state immediately.

This function returns Gst.StateChangeReturn.prototype.no_preroll if the element successfully changed its state but is not able to provide data yet. This mostly happens for live sources that only produce data in Gst.State.prototype.playing. While the state change return is equivalent to Gst.StateChangeReturn.prototype.success, it is returned to the application to signal that some sink elements might not be able to complete their state change because an element is not producing data to complete the preroll. When setting the element to playing, the preroll will complete and playback will start.

Parameters

element:

a Gst.Element to get the state of.

 int

timeout:

a Gst.ClockTime to specify the timeout for an async state change or Gst.CLOCK_TIME_NONE for infinite timeout.

 int

Returns a tuple made of

Gst.StateChangeReturn.prototype.success if the element has no more pending state and the last state change succeeded, Gst.StateChangeReturn.prototype.async if the element is still performing a state change or Gst.StateChangeReturn.prototype.failure if the last state change failed.

This function can return Gst.StateChangeReturn.prototype.async, in which case the element will perform the remainder of the state change asynchronously in another thread. Gst.Element.prototype.lost_state 

function Gst.Element.prototype.lost_state(): {
    // javascript wrapper for 'gst_element_lost_state'
}

Brings the element to the lost state. The current state of the element is copied to the pending state so that any call to Gst.Element.prototype.get_state will return Gst.StateChangeReturn.prototype.async.

An ASYNC_START message is posted. If the element was PLAYING, it will go to PAUSED. The element will be restored to its PLAYING state by the parent pipeline when it prerolls again.

This is mostly used for elements that lost their preroll buffer in the Gst.State.prototype.paused or Gst.State.prototype.playing state after a flush, they will go to their pending state again when a new preroll buffer is queued. This function can only be called when the element is currently not in error or an async state change.

This function is used internally and should normally not be called from plugins or applications.

element:

a Gst.Element the state is lost of
