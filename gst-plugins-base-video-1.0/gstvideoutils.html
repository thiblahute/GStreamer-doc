GstVideoCodecFrame

Gst.Video.VideoCodecFrame

A Gst.Video.VideoCodecFrame represents a video frame both in raw and encoded form.

Members

GstVideo.VideoCodecFrame.SystemFrameNumber (uint) –

Unique identifier for the frame. Use this if you need to get hold of the frame later (like when data is being decoded). Typical usage in decoders is to set this on the opaque value provided to the library and get back the frame using Gst.Video.VideoDecoder.GetFrame

GstVideo.VideoCodecFrame.DecodeFrameNumber (uint) –
No description available
GstVideo.VideoCodecFrame.PresentationFrameNumber (uint) –
No description available
GstVideo.VideoCodecFrame.Dts (Gst.ClockTime) –

Decoding timestamp

GstVideo.VideoCodecFrame.Pts (Gst.ClockTime) –

Presentation timestamp

GstVideo.VideoCodecFrame.Duration (Gst.ClockTime) –

Duration of the frame

GstVideo.VideoCodecFrame.DistanceFromSync (int) –

Distance in frames from the last synchronization point.

GstVideo.VideoCodecFrame.InputBuffer (Gst.Buffer) –

the input Gst.Buffer that created this frame. The buffer is owned by the frame and references to the frame instead of the buffer should be kept.

GstVideo.VideoCodecFrame.OutputBuffer (Gst.Buffer) –

the output Gst.Buffer. Implementations should set this either directly, or by using the gst_video_decoder_allocate_output_frame() or gst_video_decoder_allocate_output_buffer() methods. The buffer is owned by the frame and references to the frame instead of the buffer should be kept.

GstVideo.VideoCodecFrame.Deadline (Gst.ClockTime) –

Running time when the frame will be used.

Gst.Video.VideoCodecState

Structure representing the state of an incoming or outgoing video stream for encoders and decoders.

Decoders and encoders will receive such a state through their respective set_format vmethods.

Decoders and encoders can set the downstream state, by using the gst_video_decoder_set_output_state() or gst_video_encoder_set_output_state() methods.
