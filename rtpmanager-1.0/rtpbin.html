    // javascript callback for the 'on-bundled-ssrc' signal
}

Notify of a new incoming bundled SSRC. If no handler is connected to the signal then the rtpsession (not introspectable) created for the recv_rtp_sink_%u request pad will be managing this new SSRC. However if there is a handler connected then the application can decided to dispatch this new stream to another session by providing its ID as return value of the handler. This can be particularly useful to keep retransmission SSRCs grouped with the session for which they handle retransmission.
