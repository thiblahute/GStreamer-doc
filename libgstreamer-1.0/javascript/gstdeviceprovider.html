GstDeviceProvider

A Gst.DeviceProvider subclass is provided by a plugin that handles devices if there is a way to programatically list connected devices. It can also optionally provide updates to the list of connected devices.

Each Gst.DeviceProvider subclass is a singleton, a plugin should normally provide a single subclass for all devices.

Applications would normally use a Gst.DeviceMonitor to monitor devices from all relevant providers.

Starts providering the devices. This will cause Gst.MessageType.prototype.device_added and Gst.MessageType.prototype.device_removed messages to be posted on the provider's bus when devices are added or removed from the system.

Since the Gst.DeviceProvider is a singleton, Gst.DeviceProvider.prototype.start may already have been called by another user of the object, Gst.DeviceProvider.prototype.stop needs to be called the same number of times.
