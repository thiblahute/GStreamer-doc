GESPipeline

GES.Pipeline allows developers to view and render GES.Timeline in a simple fashion. Its usage is inspired by the 'playbin' element from gst-plugins-base.

GES.Pipeline.prototype.get_thumbnail 

function GES.Pipeline.prototype.get_thumbnail(caps: Gst.Caps): {
    // javascript wrapper for 'ges_pipeline_get_thumbnail'
}

Returns a Gst.Sample with the currently playing image in the format specified by caps. The caller should free the sample with gst_sample_unref (not introspectable) when finished. If ANY caps are specified, the information will be returned in the whatever format is currently used by the sink. This information can be retrieve from caps associated with the buffer.

function GES.Pipeline.prototype.set_mode(mode: GES.PipelineFlags): {
    // javascript wrapper for 'ges_pipeline_set_mode'
}

switches the pipeline to the specified mode. The default mode when creating a GES.Pipeline is GES.PipelineFlags.prototype.full_preview.

Note: The pipeline will be set to Gst.State.prototype.null during this call due to the internal changes that happen. The caller will therefore have to set the pipeline to the requested state after calling this method.
