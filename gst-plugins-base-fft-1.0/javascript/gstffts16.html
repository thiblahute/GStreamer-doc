GstFFTS16

GstFFTS16 provides a FFT implementation and related functions for signed 16 bit integer samples. To use this call gst_fft_s16_new (not introspectable) for allocating a GstFFTS16 instance with the appropriate parameters and then call GstFft.FFTS16.prototype.fft or GstFft.FFTS16.prototype.inverse_fft to perform the FFT or inverse FFT on a buffer of samples.

After use free the GstFFTS16 instance with GstFft.FFTS16.prototype.free.

For the best performance use GstFft.prototype.fft_next_fast_length to get a number that is entirely a product of 2, 3 and 5 and use this as the len parameter for gst_fft_s16_new (not introspectable).

The len parameter specifies the number of samples in the time domain that will be processed or generated. The number of samples in the frequency domain is len/2 + 1. To get n samples in the frequency domain use 2*n - 2 as len.

Before performing the FFT on time domain data it usually makes sense to apply a window function to it. For this GstFft.FFTS16.prototype.window can comfortably be used.

Be aware, that you can't simply run GstFft.FFTS16.prototype.inverse_fft on the resulting frequency data of GstFft.FFTS16.prototype.fft to get the original data back. The relation between them is iFFT (FFT (x)) = x / nfft where nfft is the length of the FFT. This also has to be taken into account when calculation the magnitude of the frequency data.

function GstFft.FFTS16.prototype.fft(timedata: Number, freqdata: GstFft.FFTS16Complex *): {
    // javascript wrapper for 'gst_fft_s16_fft'
}

This performs the FFT on timedata and puts the result in freqdata.

timedata must have as many samples as specified with the len parameter while allocating the GstFFTS16 instance with gst_fft_s16_new (not introspectable).

freqdata must be large enough to hold len/2 + 1 GstFft.FFTS16Complex frequency domain samples.
