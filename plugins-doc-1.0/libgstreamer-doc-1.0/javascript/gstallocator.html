GstAllocatorClass

Memory is usually created by allocators with a Gst.Allocator.prototype.alloc method call. When null is used as the allocator, the default allocator will be used.

New allocators can be registered with Gst.Allocator.prototype.register. Allocators are identified by name and can be retrieved with Gst.Allocator.prototype.find. Gst.Allocator.prototype.set_default can be used to change the default allocator.

New memory can be created with Gst.Memory.prototype.new_wrapped that wraps the memory allocated elsewhere.

Gst.Allocator.prototype.alloc 

function Gst.Allocator.prototype.alloc(size: Number, params: Gst.AllocationParams): {
    // javascript wrapper for 'gst_allocator_alloc'
}

Use allocator to allocate a new memory block with memory that is at least size big.

The optional params can specify the prefix and padding for the memory. If null is passed, no flags, no extra prefix/padding and a default alignment is used.

The prefix/padding will be filled with 0 if flags contains Gst.MemoryFlags.prototype.zero_prefixed and Gst.MemoryFlags.prototype.zero_padded respectively.

When allocator is null, the default allocator will be used.

The alignment in params is given as a bitmask so that align + 1 equals the amount of bytes to align to. For example, to align to 8 bytes, use an alignment of 7.
