GstMiniObject

Gst.MiniObject is a simple structure that can be used to implement refcounted types.

Subclasses will include Gst.MiniObject as the first member in their structure and then call gst_mini_object_init (not introspectable) to initialize the Gst.MiniObject fields.

gst_mini_object_ref (not introspectable) and gst_mini_object_unref (not introspectable) increment and decrement the refcount respectively. When the refcount of a mini-object reaches 0, the dispose function is called first and when this returns True, the free function of the miniobject is called.

A copy can be made with gst_mini_object_copy (not introspectable).

Gst.MiniObject.is_writable will return True when the refcount of the object is exactly 1, meaning the current caller has the only reference to the object. gst_mini_object_make_writable (not introspectable) will return a writable version of the object, which might be a new copy when the refcount was not 1.

Opaque data can be associated with a Gst.MiniObject with Gst.MiniObject.set_qdata and Gst.MiniObject.get_qdata. The data is meant to be specific to the particular object and is not automatically copied with gst_mini_object_copy (not introspectable) or similar methods.

A weak reference can be added and remove with gst_mini_object_weak_ref (not introspectable) and gst_mini_object_weak_unref (not introspectable) respectively.

Gst.MiniObject.is_writable 

def Gst.MiniObject.is_writable (self):
    #python wrapper for 'gst_mini_object_is_writable'

If mini_object has the LOCKABLE flag set, check if the current EXCLUSIVE lock on object is the only one, this means that changes to the object will not be visible to any other object.

If the LOCKABLE flag is not set, check if the refcount of mini_object is exactly 1, meaning that no other reference exists to the object and that the object is therefore writable.

Modification of a mini-object should only be done after verifying that it is writable.

Parameters

mini_object:

the mini-object to check

Gst.MiniObject.set_qdata 

def Gst.MiniObject.set_qdata (self, quark, data, destroy):
    #python wrapper for 'gst_mini_object_set_qdata'

This sets an opaque, named pointer on a miniobject. The name is specified through a GLib.Quark (retrieved e.g. via GLib.quark_from_static_string), and the pointer can be gotten back from the object with Gst.MiniObject.get_qdata until the object is disposed. Setting a previously set user data pointer, overrides (frees) the old pointer set, using None as pointer essentially removes the data stored.

destroy may be specified which is called with data as argument when the object is disposed, or the data is being overwritten by a call to Gst.MiniObject.set_qdata with the same quark.
