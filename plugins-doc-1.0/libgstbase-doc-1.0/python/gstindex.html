GstIndexGroup

GstIndex (not introspectable) is used to generate a stream index of one or more elements in a pipeline.

Elements will overload the set_index and get_index virtual methods in Gst.Element. When streaming data, the element will add index entries if it has an index set.

Each element that adds to the index will do that using a writer_id. The writer_id is obtained from gst_index_get_writer_id (not introspectable).

The application that wants to index the stream will create a new index object using gst_index_new or gst_index_factory_make. The index is assigned to a specific element, a bin or the whole pipeline. This will cause indexable elements to add entires to the index while playing.

                                              Gst.Object writer,
                                              int id)

Before entries can be added to the index, a writer should obtain a unique id. The methods to add new entries to the index require this id as an argument.

The application can implement a custom function to map the writer object to a string. That string will be used to register or look up an id in the index.

The caller must not hold writer's GST_OBJECT_LOCK (not introspectable), as the default resolver may call functions that take the object lock as well, and the lock is not recursive.
