GstRTSPStreamTransport

The GstRtspServer.RTSPStreamTransport configures the transport used by a GstRtspServer.RTSPStream. It is usually manages by a GstRtspServer.RTSPSessionMedia object.

With GstRtspServer.RTSPStreamTransport.prototype.set_callbacks, callbacks can be configured to handle the RTP and RTCP packets from the stream, for example when they need to be sent over TCP.

With GstRtspServer.RTSPStreamTransport.prototype.set_active the transports are added and removed from the stream.

A GstRtspServer.RTSPStream will call GstRtspServer.RTSPStreamTransport.prototype.keep_alive when RTCP is received from the client. It will also call GstRtspServer.RTSPStreamTransport.prototype.set_timed_out when a receiver has timed out.
