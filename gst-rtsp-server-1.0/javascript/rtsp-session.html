GstRTSPSession

The GstRtspServer.RTSPSession is identified by an id, unique in the GstRtspServer.RTSPSessionPool that created the session and manages media and its configuration.

A GstRtspServer.RTSPSession has a timeout that can be retrieved with GstRtspServer.RTSPSession.prototype.get_timeout. You can check if the sessions is expired with GstRtspServer.RTSPSession.prototype.is_expired. GstRtspServer.RTSPSession.prototype.touch will reset the expiration counter of the session.

When a client configures a media with SETUP, a session will be created to keep track of the configuration of that media. With GstRtspServer.RTSPSession.prototype.manage_media, the media is added to the managed media in the session. With GstRtspServer.RTSPSession.prototype.release_media the media can be released again from the session. Managed media is identified in the sessions with a url. Use GstRtspServer.RTSPSession.prototype.get_media to get the media that matches (part of) the given url.

The media in a session can be iterated with GstRtspServer.RTSPSession.prototype.filter.

GstRtspServer.RTSPSession.prototype.filter 

function GstRtspServer.RTSPSession.prototype.filter(func: GstRtspServer.RTSPSessionFilterFunc, user_data: Object): {
    // javascript wrapper for 'gst_rtsp_session_filter'
}

Call func for each media in sess. The result value of func determines what happens to the media. func will be called with sess locked so no further actions on sess can be performed from func.

If func returns GstRtspServer.RTSPFilterResult, the media will be removed from sess.

If func returns GstRtspServer.RTSPFilterResult, the media will remain in sess.

If func returns GstRtspServer.RTSPFilterResult, the media will remain in sess but will also be added with an additional ref to the result GLib.List of this function..

When func is null, GstRtspServer.RTSPFilterResult will be assumed for all media.

Parameters:

sess ( GstRtspServer.RTSPSession ) –

a GstRtspServer.RTSPSession

func ( GstRtspServer.RTSPSessionFilterFunc ) –

a callback

user_data ( Object ) –

user data passed to func

Returns ( GLib.List ) –

a GList with all media for which func returned GstRtspServer.RTSPFilterResult. After usage, each element in the GLib.List should be unreffed before the list is freed.

Callbacks

GstRtspServer.RTSPSessionFilterFunc 

GstRtspServer.RTSPFilterResult
(*GstRtspServer.RTSPSessionFilterFunc) (GstRtspServer.RTSPSession,
                                        GstRtspServer.RTSPSessionMedia,
                                        Object)

This function will be called by the GstRtspServer.RTSPSession.prototype.filter. An implementation should return a value of GstRtspServer.RTSPFilterResult.

When this function returns GstRtspServer.RTSPFilterResult, media will be removed from sess.

A return value of GstRtspServer.RTSPFilterResult will leave media untouched in sess.

A value of GST_RTSP_FILTER_REF will add media to the result GLib.List of GstRtspServer.RTSPSession.prototype.filter.

Parameters:

sess ( GstRtspServer.RTSPSession ) –

a GstRtspServer.RTSPSession object

media ( GstRtspServer.RTSPSessionMedia ) –

a GstRtspServer.RTSPSessionMedia in sess

user_data ( Object ) –

user data that has been given to GstRtspServer.RTSPSession.prototype.filter

Returns ( GstRtspServer.RTSPFilterResult ) –

a GstRtspServer.RTSPFilterResult.
